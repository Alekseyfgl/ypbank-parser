# YPBank Converter

Учебный проект по Rust для чтения, парсинга, сериализации, десериализации, конвертации и сравнения финансовых транзакций YPBank в трёх форматах:

- `YPBankCsv`
- `YPBankText`
- `YPBankBin`

Проект соответствует заданию модуля про `Read`/`Write`: библиотека работает с абстракциями стандартной библиотеки, а CLI-приложения используют эту библиотеку для чтения файлов и вывода результата.

## Состав проекта

В одном Cargo-пакете находятся три target-crate:

- библиотека `ypbank_converter`
- CLI-конвертер `ypbank_converter`
- CLI-компаратор `ypbank_compare`

## Структура

```text
Cargo.toml
src/
  lib.rs
  error.rs
  format.rs
  types.rs
  formats/
    mod.rs
    csv.rs
    text.rs
    binary.rs
  main.rs
  bin/
    ypbank_compare.rs
```

## Что умеет библиотека

- читать транзакции из любого источника, реализующего `Read`
- записывать транзакции в любой приёмник, реализующий `Write`
- конвертировать данные между форматами `csv`, `text` и `binary`
- возвращать понятные ошибки без `unwrap`
- предоставлять единый тип транзакции для всех форматов

Основные публичные точки входа:

- `read_transactions`
- `write_transactions`
- `convert`
- `read_csv` / `write_csv`
- `read_text` / `write_text`
- `read_binary` / `write_binary`

## Поддерживаемые форматы

`YPBankCsv`

- текстовый CSV с фиксированным заголовком
- описание записывается в кавычках

`YPBankText`

- текстовый формат `KEY: VALUE`
- записи разделяются пустыми строками
- комментарии с `#` игнорируются

`YPBankBin`

- бинарный формат с `MAGIC` (`YPBN`) и `RECORD_SIZE`
- многобайтовые числа кодируются в `big-endian`
- поле `DESCRIPTION` читается и пишется как UTF-8 без скрытой модификации

## Важные особенности реализации

- Библиотека не привязана к `File`: внутри парсеров используются `Read`, `Write`, `BufRead`, `BufReader`, `BufWriter`.
- При выводе в `csv` и `text` описание нормализуется к каноническому текстовому виду. Это нужно для совместимости с предоставленными примерами, где бинарный файл хранит описание с внешними кавычками.
- `ypbank_compare` сравнивает записи содержательно и при различии показывает первую несовпавшую транзакцию и конкретные отличающиеся поля.

## CLI: конвертация

Конвертер читает входной файл и пишет результат в `stdout`.

Справка по аргументам:

```bash
cargo run -- --help
```

Пример конвертации `csv -> text`:

```bash
cargo run -- \
  --input 'Примеры_файлов/records_example.csv' \
  --input-format csv \
  --output-format text
```

Пример конвертации `csv -> binary` с перенаправлением в файл:

```bash
cargo run -- \
  --input 'Примеры_файлов/records_example.csv' \
  --input-format csv \
  --output-format binary \
  > output.bin
```

Если использовать `> output.bin`, файл будет создан в текущей директории, из которой запущена команда.

Пример конвертации `text -> csv` с сохранением в файл:

```bash
cargo run -- \
  --input 'Примеры_файлов/records_example.txt' \
  --input-format text \
  --output-format csv \
  > output.csv
```

## CLI: сравнение

Компаратор читает два файла в любых поддерживаемых форматах и сообщает, совпадают ли они.

Справка по аргументам:

```bash
cargo run --bin ypbank_compare -- --help
```

Пример сравнения файлов из репозитория:

```bash
cargo run --bin ypbank_compare -- \
  --file1 'Примеры_файлов/records_example.bin' \
  --format1 binary \
  --file2 'Примеры_файлов/records_example.csv' \
  --format2 csv
```

Если записи идентичны, программа печатает:

```text
Записи транзакций в '...' и '...' совпадают.
```

Если записи отличаются, программа показывает первую несовпавшую запись и список отличающихся полей.

## Полный сценарий проверки

Сконвертировать `csv -> binary`, сохранить результат в файл и затем сравнить его с исходным CSV:

```bash
cargo run -- \
  --input 'Примеры_файлов/records_example.csv' \
  --input-format csv \
  --output-format binary \
  > output.bin
```

```bash
cargo run --bin ypbank_compare -- \
  --file1 output.bin \
  --format1 binary \
  --file2 'Примеры_файлов/records_example.csv' \
  --format2 csv
```
